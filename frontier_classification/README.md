# 한국 변새시(邊塞詩) 이항 분류 파이프라인

한국문집총간 소재 한시를 대상으로 변새시(frontier poetry) 여부를 판별하는 이항 분류 실험 코드.
규칙기반 → TF-IDF → (예정) LLM 기반 순으로 방법론을 비교한다.

## 데이터

- `data/변새시_2차정리본.txt` — **현재 사용 중인 골드 표준 데이터(255수)**. 12명 작가(고죽유고
  등)의 시가 전부 `<Poem>` XML로 마크업되어 있고(`<term>`, `<rhyme>`, `<Allusion>`,
  `<Theme category="frontier" evidence="...">`), 전부 변새시로 태깅되어 있다(= positive-only
  골드셋). 여러 하위 주제 태그(sympathy, farewell 등)가 함께 붙어 있는 multi-label 구조.
  1차정리본(223수) 대비 정두경(동명집) 32수 전체 재검토 후 신규 35편이 추가됐다 — 정두경은
  규칙기반 파일럿에서 걸림률이 가장 높았던(46%) 작가였던 만큼, 골드데이터가 놓친 진짜 변새시가
  많았다는 초기 진단과 방향이 맞았던 셈이다.
  - **알려진 제목 불일치**: 새로 추가된 정두경 시 중 8편(苦寒行, 楊經理北征歌 연작 등)은 제목이
    `0349.동명집.txt` 코퍼스의 표기와 정확히 일치하지 않는다(연작 "십수 其一~其十" 표기 차이 등).
    그래서 파일럿/TF-IDF 학습에서 "이미 아는 골드"로 제외되지 못하고 "새 후보"로 다시 걸리는
    경우가 있다 (전체 대비 미미한 수준, 8/568). 정확히 하려면 코퍼스 쪽 제목도 연작 단위로
    분리하거나 느슨한 매칭(부분 문자열)으로 바꿔야 한다.
- `data/변새시_1차정리본.txt` — 위 2차정리본의 이전 버전(223수). 비교용으로 보존.
- `data/변새시_중국_작품소거.txt` — 성당(盛唐) 변새시 대표작 245수 (잠삼·왕유·이하·고적 등).
  간체자 원문, 태그 없음. 중국 대조군.
- `../2025_munzip_title_text_ver/` (저장소 루트) — 문집총간 20개 문집, 23,526수. 그중 12개
  문집(`common.py`의 `AUTHOR_FILE`)이 골드데이터와 저자가 겹치고, 나머지 8개(`EXTRA_COLLECTION_FILE`)는
  아직 라벨이 없는 확장 적용 대상이다.
  - **주의**: 20개 중 6개는 동명이서(同名異書)로 문집총간 내 accession 번호가 여러 개 존재한다.
    파일 내용에 저자명이 없어 자동 구분이 불가능했고, 저자(사용자)가 직접 확인한 번호를 썼다
    (눌재집→0104, 동명집→0349, 소재집→0159, 지천집→0192, 현곡집→0288, 현주집→0303).

## 파이프라인

```
scripts/01_extract_thesaurus.py   골드데이터의 <term>/evidence/Allusion에서 후보 시어 추출
scripts/02_build_tiers.py         잡음 제거 + "특정(decisive)" / "범용(supporting-only)" 등급 분리
scripts/03_rule_classifier.py     규칙기반 분류기 + 골드 재현율/작가별 파일럿 테스트
scripts/04_tfidf_classifier.py    문자 n-gram TF-IDF + LogisticRegression, 5-fold 교차검증
scripts/05_score_ambiguous.py     TF-IDF 모델로 규칙기반이 걸어낸 애매한 사례들을 재심사
scripts/06_llm_classifier.py     Gemini(무료 티어)로 전체 맥락 기반 분류, 규칙/TF-IDF와 비교
```

실행은 `scripts/` 안에서 순서대로 (상대경로 기준).

## 방법론적으로 겪은 시행착오 (중요)

1. **`<term>` 태그는 순수 변새 어휘가 아니다.** 시 안에서 뭔가 강조할 때 다는 범용 emphasis
   마크업이라, 그대로 시소러스에 넣으면 不見·白日·黃昏처럼 아무 한시에나 나오는 상투어가
   섞여 들어간다. → `evidence` 속성(연구자가 실제로 "이래서 변새시다"라고 지목한 근거어)과
   교차검증된 것만 우선 신뢰하도록 설계.
2. **그래도 오탐이 심했다.** 12명 작가 전원에게 돌려보니, 아무리 불용어를 다듬어도 걸림률이
   22~58%(평균 ~35%)에서 수렴했다. 골드데이터상 실제 변새시 비율은 작가당 1~5%에 불과한데
   말이다. → 원인: 조선 문신들은 관료로서 변방 방어·인사를 일상적으로 다뤘기 때문에, 만사·
   전별시 등 변새시가 아닌 시에도 將軍·關防 같은 단어가 흔히 등장한다. **"변새 어휘 포함
   여부"만으로는 "변방 경험을 다룬 시"와 "변방 관련 인물을 스쳐 언급한 시"를 구별할 수
   없다** — 이게 규칙기반 방식의 구조적 천장이다.
3. **TF-IDF(문자 n-gram) + Logistic Regression으로 비교.** 5-fold 교차검증 기준
   정밀도 1.00 / 재현율 0.80 / F1 0.89 (`results/tfidf_vs_rule_results.txt`, 2차정리본
   255수 기준). 학습된 모델의 최상위 판별 자질이 손으로 다듬은 시소러스 핵심어(將軍·邊·塞下·
   孤城·單于·陰山 등)와 크게 겹쳐서, 서로 다른 두 방법이 같은 핵심 어휘로 수렴한다는 교차검증이
   됐다.
4. **규칙기반이 걸어낸 3,997수의 "애매한 사례"를 TF-IDF로 재심사**(`05_score_ambiguous.py`).
   TF-IDF가 동의한 건 단 28수(0.7%)뿐. 상위권을 보면 "楊經理北征歌 十首"(정두경, 연작 중 제목
   불일치로 걸린 것들 — 사실상 이미 아는 시), "塞下曲" 연작(이수광·임제), "送黃景潤爲鏡城判官"
   (임제, 함경도 국경 고을 부임 전송시) 등 **골드데이터가 놓쳤거나 제목 불일치로 빠진 진짜
   변새시**였다. 반대로 최하위권은 述懷聯句·遊頭流山百韻처럼 명백히 무관한 시였다. → TF-IDF가
   규칙기반의 오탐을 정확히 걸러내면서, 골드데이터에 없던 후보까지 찾아낸 것으로 보인다.
5. **LLM(Gemini) 기반 분류 — 골드데이터 전체(223/255수 채점 완료)** (`06_llm_classifier.py`).
   무료 티어라 분당/일별 요청 한도가 빡빡해서(모델별로 분당 5회, 일 20회 수준) 재개(resume)
   기능과 배치 처리(여러 편을 한 프롬프트에 묶어 한 번의 요청으로 처리)를 넣어서 완주했다.
   - **최종 정확도: 168/223 (75.3%)** — 초기 소표본(22수)에서 본 95.5%는 표본이 작아 우연히
     좋게 나온 수치였다. 전체로 보면 TF-IDF(재현율 0.80)보다 오히려 살짝 낮다.
   - **불일치 55건은 무작위 오류가 아니라 두 개의 인식 가능한 하위 장르에 몰려 있었다** —
     연구적으로 가장 중요한 발견:
     - **만사/애도시(哀·輓 계열, 13건)** — 임진왜란 순국 인물(조헌·고경명·김덕령·신립·
       송상현 등)을 애도하는 시. LLM은 "죽음을 애도하는 시일 뿐, 변방 생활을 다룬 시가
       아니다"로 일관되게 판단.
     - **전별시·차운시(送/次韻 계열, 약 17건)** — 변방 부임자를 전송하거나(送...赴平安道戎幕
       등), 단순히 지인과 운을 주고받는 사교시(次韻訥軒, 送李提督 연작 등). LLM은 "인사치레·
       사교 목적이 중심이지 변방 경험 자체가 중심 주제는 아니다"로 판단.
     - 이 둘을 합치면 55건 중 30건(55%)로, **"변방과 관련된 인물에게 보내거나 그를 애도하는
       시"를 변새시로 볼 것인가**라는 정의상의 경계에 체계적으로 몰려 있다. 골드데이터
       컴파일러는 "인물이 변방과 관련되면 포함"이라는 더 넓은 기준을, LLM은 "시 자체가
       변방 경험을 묘사해야 한다"는 더 좁은 기준을 쓴 것으로 보인다.
     - **결정**: 이 경계 사례들은 프롬프트를 손봐서 LLM을 골드데이터 기준에 맞추기보다(A안),
       **연구 자체에서 "변새시 개념의 경계 사례"로 다루기로 함**(B안) — 전체 불일치 목록은
       `results/llm_gold_recall_sample.txt`에서 `False`로 표시된 항목 참고.
   - 그 밖의 실패 16건은 API 오류로 재시도 가능한 항목(`[FAILED]`로 표시), 같은 명령으로
     재실행하면 자동으로 이어서 처리됨.
   - 무료 API 모델명은 자주 바뀌므로(가령 이 실험 중에도 `gemini-2.5-flash`가 신규 사용자에게
     막혔다), 실행 전 `python -c "from google import genai; ...; c.models.list()"`로 현재 계정에서
     쓸 수 있는 모델명을 먼저 확인할 것.

`results/tfidf_vs_rule_results.txt`에 있는 "규칙기반 정밀도 100%" 수치는 **순환논리로 인한
착시**임을 유의할 것 — 평가에 쓰인 negative 세트 자체가 규칙기반이 안 걸리는 시들만으로
구성되어 있어서 나온 결과다. 진짜 비교 기준은 TF-IDF/LLM 쪽 수치.

## 방법론 비교 요약

| 방법 | 재현율/정확도 | 비고 |
|---|---|---|
| 규칙기반(시소러스 매칭) | 재현율 98.8% | 정밀도는 작가당 23~58% 오탐 — 구조적 한계 |
| TF-IDF + LogisticRegression | 재현율 0.80, F1 0.89 | 오탐을 가장 잘 걸러냄, 골드 누락 후보도 발견 |
| LLM(Gemini, 전체 맥락 판단) | 정확도 75.3% | 오차가 무작위가 아니라 "만사/전별시" 장르 경계에 체계적으로 집중 — 성능 수치 자체보다 **어디서, 왜 갈리는지**가 더 중요한 결과 |
