# 한국 변새시(邊塞詩) 이항 분류 파이프라인

한국문집총간 소재 한시를 대상으로 변새시(frontier poetry) 여부를 판별하는 이항 분류 실험 코드.
규칙기반 → TF-IDF → (예정) LLM 기반 순으로 방법론을 비교한다.

## 데이터

- `data/변새시_1차정리본.txt` — 12명 작가(고죽유고 등)의 골드 표준 데이터. 223수 전부
  `<Poem>` XML로 마크업되어 있고(`<term>`, `<rhyme>`, `<Allusion>`, `<Theme category="frontier"
  evidence="...">`), 전부 변새시로 태깅되어 있다(= positive-only 골드셋). 여러 하위 주제
  태그(sympathy, farewell 등)가 함께 붙어 있는 multi-label 구조.
- `data/변새시_중국_작품소거.txt` — 성당(盛唐) 변새시 대표작 245수 (잠삼·왕유·이하·고적 등).
  간체자 원문, 태그 없음. 중국 대조군.
- `../2025_munzip_title_text_ver/` (저장소 루트) — 문집총간 20개 문집, 23,526수. 그중 12개
  문집(`common.py`의 `AUTHOR_FILE`)이 골드데이터와 저자가 겹치고, 나머지 8개(`EXTRA_COLLECTION_FILE`)는
  아직 라벨이 없는 확장 적용 대상이다.
  - **주의**: 20개 중 6개는 동명이서(同名異書)로 문집총간 내 accession 번호가 여러 개 존재한다.
    파일 내용에 저자명이 없어 자동 구분이 불가능했고, 저자(사용자)가 직접 확인한 번호를 썼다
    (눌재집→0104, 동명집→0349, 소재집→0159, 지천집→0192, 현곡집→0288, 현주집→0303).

## 파이프라인

```
scripts/01_extract_thesaurus.py   골드데이터의 <term>/evidence/Allusion에서 후보 시어 추출
scripts/02_build_tiers.py         잡음 제거 + "특정(decisive)" / "범용(supporting-only)" 등급 분리
scripts/03_rule_classifier.py     규칙기반 분류기 + 골드 재현율/작가별 파일럿 테스트
scripts/04_tfidf_classifier.py    문자 n-gram TF-IDF + LogisticRegression, 5-fold 교차검증
scripts/05_score_ambiguous.py     TF-IDF 모델로 규칙기반이 걸어낸 애매한 사례들을 재심사
```

실행은 `scripts/` 안에서 순서대로 (상대경로 기준).

## 방법론적으로 겪은 시행착오 (중요)

1. **`<term>` 태그는 순수 변새 어휘가 아니다.** 시 안에서 뭔가 강조할 때 다는 범용 emphasis
   마크업이라, 그대로 시소러스에 넣으면 不見·白日·黃昏처럼 아무 한시에나 나오는 상투어가
   섞여 들어간다. → `evidence` 속성(연구자가 실제로 "이래서 변새시다"라고 지목한 근거어)과
   교차검증된 것만 우선 신뢰하도록 설계.
2. **그래도 오탐이 심했다.** 12명 작가 전원에게 돌려보니, 아무리 불용어를 다듬어도 걸림률이
   22~58%(평균 ~35%)에서 수렴했다. 골드데이터상 실제 변새시 비율은 작가당 1~5%에 불과한데
   말이다. → 원인: 조선 문신들은 관료로서 변방 방어·인사를 일상적으로 다뤘기 때문에, 만사·
   전별시 등 변새시가 아닌 시에도 將軍·關防 같은 단어가 흔히 등장한다. **"변새 어휘 포함
   여부"만으로는 "변방 경험을 다룬 시"와 "변방 관련 인물을 스쳐 언급한 시"를 구별할 수
   없다** — 이게 규칙기반 방식의 구조적 천장이다.
3. **TF-IDF(문자 n-gram) + Logistic Regression으로 비교.** 5-fold 교차검증 기준
   정밀도 0.99 / 재현율 0.79 / F1 0.87 (`results/tfidf_vs_rule_results.txt`). 학습된 모델의
   최상위 판별 자질이 손으로 다듬은 시소러스 핵심어(將軍·邊·塞下·孤城·單于·陰山 등)와 크게
   겹쳐서, 서로 다른 두 방법이 같은 핵심 어휘로 수렴한다는 교차검증이 됐다.
4. **규칙기반이 걸어낸 3,895수의 "애매한 사례"를 TF-IDF로 재심사**(`05_score_ambiguous.py`).
   TF-IDF가 동의한 건 단 17수(0.4%)뿐. 그런데 그 17수 + 상위권을 직접 보면 "塞下曲" 연작,
   "送...赴平安道戎幕"(평안도 군막 부임 전송시), "次受降亭韻 滿浦"(실제 압록강변 국경 진지)
   등 **골드데이터가 "1차정리본"이라는 이름 그대로 놓친 진짜 변새시로 보이는 시들**이었다.
   반대로 최하위권은 述懷聯句·遊頭流山百韻처럼 명백히 무관한 시였다. → TF-IDF가 규칙기반의
   오탐을 정확히 걸러내면서, 골드데이터에 없던 후보까지 찾아낸 것으로 보인다.

`results/tfidf_vs_rule_results.txt`에 있는 "규칙기반 정밀도 100%" 수치는 **순환논리로 인한
착시**임을 유의할 것 — 평가에 쓰인 negative 세트 자체가 규칙기반이 안 걸리는 시들만으로
구성되어 있어서 나온 결과다. 진짜 비교 기준은 TF-IDF 쪽 수치.
