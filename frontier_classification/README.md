# 한국 변새시(邊塞詩) 이항 분류 파이프라인

한국문집총간 소재 한시를 대상으로 변새시(frontier poetry) 여부를 판별하는 이항 분류 실험 코드.
규칙기반 → TF-IDF → (예정) LLM 기반 순으로 방법론을 비교한다.

## 데이터

- `data/변새시_2차정리본.txt` — **현재 사용 중인 골드 표준 데이터(255수)**. 12명 작가(고죽유고
  등)의 시가 전부 `<Poem>` XML로 마크업되어 있고(`<term>`, `<rhyme>`, `<Allusion>`,
  `<Theme category="frontier" evidence="...">`), 전부 변새시로 태깅되어 있다(= positive-only
  골드셋). 여러 하위 주제 태그(sympathy, farewell 등)가 함께 붙어 있는 multi-label 구조.
  1차정리본(223수) 대비 정두경(동명집) 32수 전체 재검토 후 신규 35편이 추가됐다 — 정두경은
  규칙기반 파일럿에서 걸림률이 가장 높았던(46%) 작가였던 만큼, 골드데이터가 놓친 진짜 변새시가
  많았다는 초기 진단과 방향이 맞았던 셈이다.
  - **제목 불일치 → 해결됨**: 골드데이터는 연작시를 대표 제목 하나로만 기록하는데(예: "楊經理
    北征歌"), 코퍼스는 "楊經理北征歌 十首", "...其三" 처럼 개별 수록되어 있어 완전일치 매칭으로는
    "새 후보"로 잘못 다시 걸리는 문제가 있었다. `common.title_in_gold()`가 골드 제목 뒤에
    연작 표기(十首/百首/其N)나 지명·인명 부기(예: "元帥臺 在磨天嶺", "送黃景潤爲鏡城判官 名澯")가
    붙은 경우까지 같은 시로 인식하도록 고쳤다. 12명 작가 전원에서 총 200여 건이 재분류됐고,
    수작업으로 21건을 표본 검증해 전부 정당한 매칭임을 확인했다(오탐 0건).
- `data/변새시_1차정리본.txt` — 위 2차정리본의 이전 버전(223수). 비교용으로 보존.
- `data/변새시_중국_작품소거.txt` — 성당(盛唐) 변새시 대표작 245수 (잠삼·왕유·이하·고적 등).
  간체자 원문, 태그 없음. 중국 대조군.
- `../2025_munzip_title_text_ver/` (저장소 루트) — 문집총간 20개 문집, 23,526수. 그중 12개
  문집(`common.py`의 `AUTHOR_FILE`)이 골드데이터와 저자가 겹치고, 나머지 8개(`EXTRA_COLLECTION_FILE`)는
  아직 라벨이 없는 확장 적용 대상이다.
  - **주의**: 20개 중 6개는 동명이서(同名異書)로 문집총간 내 accession 번호가 여러 개 존재한다.
    파일 내용에 저자명이 없어 자동 구분이 불가능했고, 저자(사용자)가 직접 확인한 번호를 썼다
    (눌재집→0104, 동명집→0349, 소재집→0159, 지천집→0192, 현곡집→0288, 현주집→0303).

## 파이프라인

```
scripts/01_extract_thesaurus.py   골드데이터의 <term>/evidence/Allusion에서 후보 시어 추출
scripts/02_build_tiers.py         잡음 제거 + "특정(decisive)" / "범용(supporting-only)" 등급 분리
scripts/03_rule_classifier.py     규칙기반 분류기 + 골드 재현율/작가별 파일럿 테스트
scripts/04_tfidf_classifier.py    문자 n-gram TF-IDF + LogisticRegression, 5-fold 교차검증
scripts/05_score_ambiguous.py     TF-IDF 모델로 규칙기반이 걸어낸 애매한 사례들을 재심사
scripts/06_llm_classifier.py     Gemini(무료 티어)로 전체 맥락 기반 분류, 규칙/TF-IDF와 비교
scripts/07_apply_to_extra_collections.py  검증된 TF-IDF 모델을 8개 미분류 문집에 확대 적용
scripts/08_score_full_corpus.py  문집총간 전체(651개 문집)를 규칙기반+TF-IDF로 로컬 스코어링
scripts/09_llm_verify_shortlist.py  상위 후보 숏리스트를 LLM으로 배치 검증 (무료 API, 재개 지원)
scripts/10_naive_bayes_svm_classifier.py  나이브 베이즈·선형 SVM 추가 비교 + 시별 판정근거 파일 생성
```

실행은 `scripts/` 안에서 순서대로 (상대경로 기준).

## 방법론적으로 겪은 시행착오 (중요)

1. **`<term>` 태그는 순수 변새 어휘가 아니다.** 시 안에서 뭔가 강조할 때 다는 범용 emphasis
   마크업이라, 그대로 시소러스에 넣으면 不見·白日·黃昏처럼 아무 한시에나 나오는 상투어가
   섞여 들어간다. → `evidence` 속성(연구자가 실제로 "이래서 변새시다"라고 지목한 근거어)과
   교차검증된 것만 우선 신뢰하도록 설계.
2. **그래도 오탐이 심했다.** 12명 작가 전원에게 돌려보니, 아무리 불용어를 다듬어도 걸림률이
   22~58%(평균 ~35%)에서 수렴했다. 골드데이터상 실제 변새시 비율은 작가당 1~5%에 불과한데
   말이다. → 원인: 조선 문신들은 관료로서 변방 방어·인사를 일상적으로 다뤘기 때문에, 만사·
   전별시 등 변새시가 아닌 시에도 將軍·關防 같은 단어가 흔히 등장한다. **"변새 어휘 포함
   여부"만으로는 "변방 경험을 다룬 시"와 "변방 관련 인물을 스쳐 언급한 시"를 구별할 수
   없다** — 이게 규칙기반 방식의 구조적 천장이다.
3. **TF-IDF(문자 n-gram) + Logistic Regression으로 비교.** 5-fold 교차검증 기준
   정밀도 0.996 / 재현율 0.80 / F1 0.89 (`results/tfidf_vs_rule_results.txt`, 2차정리본
   255수 + 제목 매칭 수정 반영 기준). 학습된 모델의 최상위 판별 자질이 손으로 다듬은 시소러스
   핵심어(將軍·邊·塞下·孤城·單于·陰山 등)와 크게 겹쳐서, 서로 다른 두 방법이 같은 핵심 어휘로
   수렴한다는 교차검증이 됐다.
4. **규칙기반이 걸어낸 "애매한 사례"를 TF-IDF로 재심사**(`05_score_ambiguous.py`). 제목 매칭
   수정 전에는 3,997수 중 28수(0.7%)가 TF-IDF 동의를 받았는데, 그중 상당수가 실은 연작
   제목 불일치로 인한 "가짜 신규"였다. 매칭 수정 후에는 애매 사례 풀이 3,807수로 줄고, TF-IDF가
   동의하는 건 **6수(0.2%)**뿐 — 훨씬 신뢰도 높은 후보 목록이 남았다. 상위권: 임제 "塞下曲"
   (본인 명의로는 골드에 없었음), 이행의 "送李希剛...赴平安道戎幕" 연작(부임 전송시), 최경창의
   "因李益之北歸。寄朴觀察民獻" 연작, 차천로 "弔李提督"(다른 연작인 送李提督과는 별개),
   황정욱 "哭金德源，白富寧，李先達戰死"(전사자 애도시) 등 — 전부 골드데이터에 실제로 없는
   작가·제목이라 **진짜 신규 후보**로 볼 수 있다. 최하위권은 여전히 述懷聯句·遊頭流山百韻처럼
   명백히 무관한 시였다.
5. **LLM(Gemini) 기반 분류 — 골드데이터 전체(223/255수 채점 완료)** (`06_llm_classifier.py`).
   무료 티어라 분당/일별 요청 한도가 빡빡해서(모델별로 분당 5회, 일 20회 수준) 재개(resume)
   기능과 배치 처리(여러 편을 한 프롬프트에 묶어 한 번의 요청으로 처리)를 넣어서 완주했다.
   - **최종 정확도: 168/223 (75.3%)** — 초기 소표본(22수)에서 본 95.5%는 표본이 작아 우연히
     좋게 나온 수치였다. 전체로 보면 TF-IDF(재현율 0.80)보다 오히려 살짝 낮다.
   - **불일치 55건은 무작위 오류가 아니라 두 개의 인식 가능한 하위 장르에 몰려 있었다** —
     연구적으로 가장 중요한 발견:
     - **만사/애도시(哀·輓 계열, 13건)** — 임진왜란 순국 인물(조헌·고경명·김덕령·신립·
       송상현 등)을 애도하는 시. LLM은 "죽음을 애도하는 시일 뿐, 변방 생활을 다룬 시가
       아니다"로 일관되게 판단.
     - **전별시·차운시(送/次韻 계열, 약 17건)** — 변방 부임자를 전송하거나(送...赴平安道戎幕
       등), 단순히 지인과 운을 주고받는 사교시(次韻訥軒, 送李提督 연작 등). LLM은 "인사치레·
       사교 목적이 중심이지 변방 경험 자체가 중심 주제는 아니다"로 판단.
     - 이 둘을 합치면 55건 중 30건(55%)로, **"변방과 관련된 인물에게 보내거나 그를 애도하는
       시"를 변새시로 볼 것인가**라는 정의상의 경계에 체계적으로 몰려 있다. 골드데이터
       컴파일러는 "인물이 변방과 관련되면 포함"이라는 더 넓은 기준을, LLM은 "시 자체가
       변방 경험을 묘사해야 한다"는 더 좁은 기준을 쓴 것으로 보인다.
     - **결정**: 이 경계 사례들은 프롬프트를 손봐서 LLM을 골드데이터 기준에 맞추기보다(A안),
       **연구 자체에서 "변새시 개념의 경계 사례"로 다루기로 함**(B안) — 전체 불일치 목록은
       `results/llm_gold_recall_sample.txt`에서 `False`로 표시된 항목 참고.
   - 그 밖의 실패 16건은 API 오류로 재시도 가능한 항목(`[FAILED]`로 표시), 같은 명령으로
     재실행하면 자동으로 이어서 처리됨.
   - 무료 API 모델명은 자주 바뀌므로(가령 이 실험 중에도 `gemini-2.5-flash`가 신규 사용자에게
     막혔다), 실행 전 `python -c "from google import genai; ...; c.models.list()"`로 현재 계정에서
     쓸 수 있는 모델명을 먼저 확인할 것.

`results/tfidf_vs_rule_results.txt`에 있는 "규칙기반 정밀도 100%" 수치는 **순환논리로 인한
착시**임을 유의할 것 — 평가에 쓰인 negative 세트 자체가 규칙기반이 안 걸리는 시들만으로
구성되어 있어서 나온 결과다. 진짜 비교 기준은 TF-IDF/LLM 쪽 수치.

6. **8개 미분류 문집으로 확대 적용**(`07_apply_to_extra_collections.py`, `results/
   extra_collections_scored.txt`). 12명 골드 작가로만 학습한 TF-IDF 모델을 기재집·동악집·
   성소부부고·소재집·손곡시집·읍취헌유고·현주집·호음잡고에 그대로 돌려봤다. 8개 전부 확신도
   0.5를 넘는 시는 없었다 — 예상된 결과다(학습에 쓰인 12명과 다른 작가라 문체 유사도가
   낮아서 확신도가 전반적으로 낮게 나옴). 다만 0.3~0.45대에서 걸러진 후보들을 보면:
   - **동악집**(이안눌 추정)에서 "金將軍應河挽詞"(전사장군 애도), "出塞曲", "奉陪李節度使...
     北巡邊城"(북방 변방 순찰 동행시), "廣寧行"(만주 국경 도시), "元帥臺 臺在鏡城府南海上"
     (함경도 국경 고을) 등 설득력 있는 후보가 다수 나왔다 — **12명 골드 작가 외에 새로 편입할
     만한 작가**로 보인다.
   - **손곡시집**(이달 추정)에서도 "塞下曲。贈柳摠戎", "出塞曲" 연작, "送別柳摠戎" 등이 나왔다.
   - **호음잡고**의 "傷滿浦節使沈讓鄕戰歿"(압록강변 만포 국경 진지에서 전사한 장수 애도)은
     단독으로도 눈에 띄는 후보.
   - 규칙기반 걸림 건수(기재집 420건 ~ 동악집 1,677건)는 여전히 매우 높다 — 12명 파일럿과
     같은 이유(범용 관료 어휘)로, 이 단계에서는 규칙기반 단독 수치보다 TF-IDF 확신도 순위를
     기준으로 검토하는 게 낫다.
7. **애매 사례 풀(3,807수)에서 무작위 40수를 LLM에도 태워 TF-IDF와 비교**
   (`--mode ambiguous`, `results/llm_ambiguous_sample.txt`). 결정적인 차이가 나왔다 —
   **TF-IDF는 이 풀 전체에서 0.2%(6/3,807)만 긍정으로 봤는데, 같은 풀에서 무작위로 뽑은
   40수만 해도 LLM은 22.5%(9/40)를 긍정으로 판단했다.** True로 판단한 것들의 근거를 보면
   납득할 만하다 — 정두경 "觀向化射獵"(귀화 여진족 사냥, 陰山·射鵰人), 차천로 "奉送洪鶴谷
   充賀聖節使朝京"(장성·갈석산·요동벌판·산해관), 권필 "切切何切切"(변방 간 남편을 둔 아내의
   애환 — 정부원征婦怨 계열), 이수광 "路遇聖節使洪僉知輝世..."(요동 벌판·해관) 등. 흥미롭게도
   이수광의 "送李正郞...赴統營"(통영 부임 전송)도 True로 나와, "변새"를 북방에 한정할지 남해
   수군 방어까지 포함할지의 경계 질문이 여기서도 재확인됐다.
   - **결론**: 골드 재현율만 보면 TF-IDF(0.80)와 LLM(0.753)이 비슷했지만, **라벨 없는 새
     후보를 찾아내는 발견력은 LLM이 TF-IDF보다 압도적으로 우수하다.** TF-IDF는 학습 데이터
     (255수, 12명)와 조금만 문체가 달라도 놓치는 경향이 강해, 코퍼스 확대 적용 단계에서는
     TF-IDF 단독보다 LLM(또는 LLM 재심사를 곁들인 파이프라인)을 주력으로 쓰는 게 나아 보인다.
8. **문집총간 전체(651개 문집, 368,445수)로 확대 적용**(`08_score_full_corpus.py`,
   `09_llm_verify_shortlist.py`). 로컬 스코어링(규칙기반+TF-IDF)은 전체 코퍼스에 31.6초밖에
   안 걸린다. 다만 같은 제목(특히 塞下曲·出塞曲·從軍行 같은 악부제)을 수십 명의 다른 작가가
   재사용하므로, 골드 배제는 **12명 작가 파일 안에서만** 적용하고 나머지 639개 문집은 전부
   후보로 유지했다 — 실제로 塞下曲 연작이 백곡집·난설헌시집·하곡집·해옹고 등 12명 밖의
   여러 작가에서 두루 발견됐다. TF-IDF>=0.5는 전체의 0.02%(57건)뿐이었지만 상위 1,000건
   숏리스트는 규칙기반 걸림과 겹치는 항목이 많아 유망해 보인다. LLM 검증은 무료 API 하루
   쿼터 때문에 여러 세션에 걸쳐 진행 중 (`results/full_corpus_llm_verified.txt`,
   진행 상황은 파일의 성공/실패 줄 수로 확인 가능; 재실행 시 자동 재개).
9. **나이브 베이즈·선형 SVM 추가 비교**(`10_naive_bayes_svm_classifier.py`,
   `results/nb_svm_cv_results.txt`). 제안서가 언급한 "전통적 머신러닝" 비교를 완성하기 위해
   TF-IDF+LogisticRegression 외에 두 방법을 더 추가했다. **둘 다 액면 그대로 믿으면 안 되는
   결과가 나왔다**:
   - **나이브 베이즈**: 5-fold 재현율 **7.5%**로 매우 저조. 학습 데이터(255수) 자체에서도
     110/255(43%)만 맞혔다. "觀向化射獵"(정두경)처럼 陰山·射鵰 같은 핵심 변새 시어를 "근거
     자질"로는 정확히 짚어내고도 최종 판정은 "아님"(확률 0.005)으로 나왔다 — 클래스 불균형
     (부정 7,108 : 긍정 255)에 나이브 베이즈가 특히 취약함을 보여준다.
   - **선형 SVM**: 5-fold 정밀도/재현율/F1 전부 **0.996**로 거의 완벽했지만, 이는 **과적합
     착시**다. 9만여 차원의 n-gram 특징 대비 학습 표본(255수)이 너무 적어, 학습 데이터
     255/255는 100% 맞히면서도(사실상 암기) `llm_ambiguous_sample.txt`와 같은 애매 사례
     40수에 적용하면 楡塞·沙場·陰山 같은 명백한 변새 시어가 있는 시조차 전부 "아님"(확률
     0.000~0.032)으로 오판했다.
   - **의의**: 교차검증 지표가 완벽에 가까워도(SVM 0.996) 실제 발견력은 전무할 수 있다는 걸
     보여주는 명확한 사례 — LLM이 같은 조건에서 재현율 0.753이면서도 미분류 코퍼스에서
     22.5%를 발견해낸 것과 대비하면, **단일 지표(교차검증 F1 등)만으로 방법을 고르면 안 되고
     반드시 분포 밖(out-of-distribution) 데이터로 검증해야 한다**는 방법론적 교훈이 된다.
   - `results/nb_gold_recall_sample.txt`, `svm_gold_recall_sample.txt`는 학습에 쓰인 바로 그
     255수에 대한 판정이라 **일반화 성능이 아니라 암기 성능**을 보여준다는 점에 유의 (반면
     LLM의 `llm_gold_recall_sample.txt`는 LLM이 이 데이터로 학습된 적이 없으므로 정당한 시험).
     `nb_ambiguous_sample.txt`/`svm_ambiguous_sample.txt`가 진짜 일반화 성능을 보여주는 파일.

## 방법론 비교 요약

| 방법 | 골드 재현율/정확도 | 미분류 코퍼스 발견력 | 비고 |
|---|---|---|---|
| 규칙기반(시소러스 매칭) | 재현율 98.8% | 낮음(오탐 다수) | 정밀도는 작가당 23~58% 오탐 — 구조적 한계 |
| 나이브 베이즈 | 재현율 0.075 | 사실상 0 | 클래스 불균형에 취약, 학습셋 자체도 43%만 적중 |
| TF-IDF + LogisticRegression | 재현율 0.80, F1 0.89 | **매우 낮음**(애매 풀의 0.2%만 긍정) | 골드와 비슷한 시는 잘 잡지만 문체가 다르면 놓침 |
| 선형 SVM | 재현율 0.996(교차검증) | **사실상 0**(애매 사례 40수 전부 오판) | 교차검증 수치는 과적합 착시 — 미검증 데이터에 전혀 안 통함 |
| LLM(Gemini, 전체 맥락 판단) | 정확도 75.3% | **높음**(같은 풀에서 22.5% 긍정) | 오차가 "만사/전별시" 장르 경계에 체계적으로 집중 — 발견 단계에서는 가장 실용적 |
